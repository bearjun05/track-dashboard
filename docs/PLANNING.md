# 트랙 운영 관리 대시보드 - 기획문서

## 1. 프로젝트 배경 및 목적

### 배경

교육 트랙 운영에서 학습관리매니저(학관매)는 알바 형태로 투입되어 교체가 잦다. 누가 와도 일정한 품질을 유지하려면, 업무를 표준화된 Task로 정의하고 자동으로 배정하여 팔로업이 가능한 구조가 필요하다.

동시에 운영기획매니저(운기매)는 정규직으로서 4~5개 트랙을 담당하며, 세부 업무에 직접 관여하기보다 **핵심 지표만 보고도 트랙이 정상 운영되고 있는지 판단**할 수 있어야 한다.

### 목적

1. **학관매 품질 표준화**: 정의된 Task를 자동 배정하여, 인력 교체와 무관하게 일관된 운영 품질 유지
2. **운기매 리소스 절감**: 3가지 핵심 지표만으로 트랙 상태를 파악할 수 있게 하여, 운기매가 품질 고도화에 집중할 수 있는 환경 구축
3. **체계적 팔로업**: Task별/이슈별 채팅을 통한 실시간 소통과 이력 관리

## 2. 역할 체계

```
운기매 (정규직, 1명)
 ├── 운영매 A (계약직)
 │    ├── 학관매 1 (알바)
 │    ├── 학관매 2 (알바)
 │    └── 학관매 3 (알바)
 ├── 운영매 B (계약직)
 │    ├── 학관매 4 (알바)
 │    ├── 학관매 5 (알바)
 │    └── 학관매 6 (알바)
 └── 운영매 C (계약직)
      ├── 학관매 7 (알바)
      ├── 학관매 8 (알바)
      └── 학관매 9 (알바)
```

### 운기매 (운영기획매니저)

- **고용 형태**: 정규직
- **관리 범위**: 4~5개 트랙, 운영매만 직접 관리 (학관매를 직접 관리하지 않음)
- **핵심 업무**: 3가지 지표를 통한 트랙 상태 모니터링, 운영매 업무 지시/팔로업
- **핵심 지표**:
  1. 업무완료율 (학관매) - 학관매들의 오늘 Task 완료 비율
  2. 이슈처리율 - 발생한 이슈 중 처리 완료된 비율
  3. 업무완료율 (운영매) - 운영매에게 요청한 업무의 완료 비율

### 운영매 (운영매니저)

- **고용 형태**: 계약직
- **관리 범위**: 학관매 3명 (트랙 1~2개)
- **핵심 업무**:
  - 학관매 업무 상세 관리 (배정, 확인, 팔로업)
  - 미배정 Task 수동 배정 / 신규 Task 생성
  - 학관매 휴가/부재 시 Task 재배정 또는 직접 수행
  - 운기매 지시사항 수행 및 보고
  - 학관매 이슈 1차 처리

### 학관매 (학습관리매니저)

- **고용 형태**: 알바
- **업무 범위**: 시스템이 자동 배정한 Task 수행 (출석체크, 팀순회, 수강생 관리 등)
- **핵심 특성**: 개인 업무관리 페이지에서 오늘 할 일 확인 -> 수행 -> 완료 처리

## 3. Task 자동 배정 시스템

### 3.1 Task 유형

| 유형 | 설명 | 예시 |
|------|------|------|
| **일일 반복 (daily)** | 매일 동일 시간에 자동 생성 | 출석체크(09:00), 오전 팀순회(10:00), 오후 팀순회(14:00) |
| **마일스톤 (milestone)** | 트랙 일정 기준 특정 시점에 자동 생성 | 중간평가 준비(D-7), OT 준비(D-3), 최종평가 준비(D-5) |
| **수동 (manual)** | 운영매가 직접 생성하여 배정 | 특별 면담, 추가 수강생 관리, 긴급 이슈 대응 |

### 3.2 Task 생애주기

```
[템플릿 정의] → [트랙 생성 시 자동 복제] → [일정에 따라 Task 생성]
                                               ↓
                                    [학관매 업무관리 페이지에 노출]
                                               ↓
                                    [학관매 수행 → 완료 처리]
                                               ↓
                              [미완료 시 기한 초과(overdue)로 표시]
```

### 3.3 예외 처리 (운영매 RnR)

| 상황 | 처리 방법 | 담당 |
|------|-----------|------|
| 학관매 휴가 | 다른 학관매에게 재배정 또는 운영매 직접 수행 | 운영매 |
| 시스템 미배정 Task | 운영매가 수동으로 학관매에게 배정 | 운영매 |
| 신규 업무 발생 | 운영매가 Task 생성 후 학관매에게 배정 | 운영매 |
| Task 기한 초과 | 운영매에게 알림, 재배정 또는 에스컬레이션 | 운영매 |

## 4. 페이지 구조

### 4.1 운기매 페이지 (/admin)

```
/admin                                    # 운기매 홈
├── 칸반보드 (운영매 요청사항 & 업무)
├── 기수별 실시간 채팅
└── 담당 트랙 카드 목록

/admin/tracks/[id]                        # 트랙 상세
├── 오늘의 요약 (3가지 핵심 지표)
├── 학관매별 Task 진행 현황 (테이블)
├── Task 상태별 분류 뷰 (전체/진행중/완료/미완료/미배정)
├── 미배정 Task 목록
└── 수강생 현황

/admin/tracks/[id]/staff/[staffId]        # 학관매 상세
├── Task별 대화 스레드
├── 이슈/요청 목록
└── 업무 리스트 (전체/완료/미완료)

/admin/operators/[id]                     # 운영매 상세
├── 운영매 업무 현황 (미완료/완료)
└── 트랙별 학관매 리스트
```

### 4.2 운영매 페이지 (/operator)

```
/operator                                 # 운영매 홈
├── 핵심 지표 (담당 학관매 업무완료율, 이슈처리율, 본인 업무완료율)
├── 칸반보드 (운기매 지시 업무 / 운기매에게 요청한 업무)
├── 학관매별 업무 현황 카드
└── 이슈 목록

/operator/tracks/[id]                     # 운영매 - 트랙 상세
├── 학관매별 Task 진행 현황
├── 미배정 Task 관리 (배정/재배정)
└── Task 신규 생성

/operator/staff/[staffId]                 # 운영매 - 학관매 상세
├── Task별 대화 스레드
├── 이슈/요청 목록
└── 업무 리스트 + 배정/상태 변경 기능
```

### 4.3 학관매 페이지 (/)

```
/                                         # 학관매 개인 업무관리
├── 오늘의 Task 리스트 (시간순)
├── 주간 캘린더
├── 공지사항
└── 이슈 등록
```

## 5. 채팅 구조

| 채팅 유형 | 위치 | 참여자 | 설명 |
|-----------|------|--------|------|
| **Task별 채팅** | 학관매 상세 페이지 | 학관매 ↔ 운영매 | 각 Task에 스레드 형태로 대화, Task 수행 중 질문/보고 |
| **이슈별 채팅** | 학관매 상세 / 이슈 모달 | 학관매 ↔ 운영매 | 이슈 요청 건마다 별도 대화, 상태 변경 포함 |
| **칸반 카드별 채팅** | 운기매/운영매 홈 | 운기매 ↔ 운영매 | 업무 요청/지시 건마다 채팅, 칸반 상태와 연동 |

## 6. 데이터 모델

### 6.1 Task (확장)

```typescript
interface TrackTask {
  id: string
  title: string
  description?: string
  type: 'daily' | 'milestone' | 'manual'
  templateId?: string
  trackId: string
  assigneeId?: string       // 담당 학관매 ID (없으면 미배정)
  assigneeName?: string
  status: 'pending' | 'in-progress' | 'completed' | 'overdue' | 'unassigned'
  scheduledDate: string     // 예정 일자 (YYYY-MM-DD)
  scheduledTime?: string    // 예정 시간 (HH:mm)
  dueTime?: string          // 마감 시간 (HH:mm)
  completedAt?: string
  unassignedReason?: string // 미배정 사유
  messages: TaskMessage[]   // Task별 채팅
}
```

### 6.2 Task 템플릿

```typescript
interface TaskTemplate {
  id: string
  title: string
  description?: string
  type: 'daily' | 'milestone'
  scheduledTime?: string          // daily: 매일 이 시간에 생성
  milestoneDayOffset?: number     // milestone: 트랙 시작일 기준 D+N
  dueOffsetMinutes?: number       // 생성 후 N분 후 마감
  autoAssign: boolean             // true면 학관매에게 자동 배정
}
```

## 7. 향후 고도화 방향

- 백엔드 API 연동 (현재 목데이터 + Zustand)
- 알림 시스템 (Task 기한 임박, 이슈 발생, 미완료 알림)
- 학관매 성과 리포트 / 주간·월간 통계
- Task 템플릿 에디터 (운기매가 직접 템플릿 편집)
- 모바일 대응 (학관매 업무관리 페이지)
