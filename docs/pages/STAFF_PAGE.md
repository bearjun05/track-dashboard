# 학관(Staff) 대시보드 페이지 정책

> 경로: `/staff/[id]`
> 접근 권한: 해당 학관매 본인만 접근 가능 (운기매/운영매는 접근 불가)
> 데이터 소스: `useAdminStore` (진행률 통계), `useStaffDashboard` 훅 (Task/일정 필터링), `useInterviewStore` (면담)

---

## 1. 페이지 레이아웃

### 1.1 탭 구조

| 탭 | key | 설명 |
|----|-----|------|
| 업무 | `task` | 메인 탭 — 선택 날짜의 할 일 + 운영매니저 소통 |
| 일정 | `schedule` | 캘린더 + 다가오는 주요 업무 (전체 너비) |
| 면담 | `interview` | 수강생 면담 관리 (기존) |

### 1.2 헤더

**좌측 영역**: 탭 토글(업무/일정/면담) + 구분선 + 날짜 네비게이션
**우측 영역**: 진행률 표시(업무/일정 탭) 또는 면담 통계(면담 탭) + 업무 생성 버튼(업무/일정 탭)

**날짜 네비게이션**:
- 좌/우 화살표(`ChevronLeft`/`ChevronRight`)로 이전/다음 날짜 이동
- 날짜 영역 클릭 시 캘린더 팝업(월간 그리드)으로 원하는 날짜 직접 선택
- 현재 날짜(`TODAY_STR` — `lib/date-constants.ts`에서 동적 계산)가 아닐 때 "오늘" 버튼 표시
- 선택한 날짜에 따라 업무 탭의 시간 지정 업무/마감 업무/소통 채널이 모두 해당 날짜 기준으로 필터링
- `selectedDate` 상태는 `page.tsx`에서 관리하며, `DashboardHeader`, `TimePanel`, `TodayPanel`, `CommChannel`에 전달
- 헤더 진행률은 `useAdminStore.trackTasks`에서 `staffId + selectedDate`로 필터링하여 계산 (이전 `useDashboardStore` 제거됨)
- `trackName` prop으로 소속 트랙명 동적 표시 (`plannerTracks`에서 조회)

**제거된 항목** (관리자는 사이드바를 통해 접근, 알림은 소통 채널 내 알림 탭에서 제공):
- 관리자 Link 버튼 (`Shield` 아이콘 + "관리자")
- `NotificationDropdown` 컴포넌트

### 1.3 업무 탭 (메인)

| 영역 | 비율 | 설명 |
|------|------|------|
| 시간 지정 업무 | 25% | 선택 날짜의 시간이 지정된 task를 타임라인 형태로 표시 |
| 업무 리스트 | 25% | 선택 날짜의 시간 미지정 task (요청한 / 받은 요청 / 자동 배정 / 내가 추가한) |
| 소통 채널 | 50% | 헤더 탭으로 대화 상대 전환 (사이드바 제거). 알림/운영매니저/동료 학관 |

### 1.4 패널 디자인 (macOS 스타일)

모든 패널은 동일한 macOS 스타일 카드 디자인을 사용한다:

- 컨테이너: `rounded-xl border border-foreground/[0.08] bg-background shadow-[0_1px_4px_rgba(0,0,0,0.06)]`
- 헤더 높이: `h-11` (44px), 아이콘(`h-3.5`) + 타이틀(`text-[13px] font-semibold`)
- 패널 간 간격: `gap-2` (8px)
- 전체 영역 패딩: `p-2` (8px) — 헤더, 좌우, 하단 모두 동일 여백

### 1.5 일정 탭

- 2-column 레이아웃: 캘린더(65%) + 오른쪽 패널(35%)
- 뷰 전환: [월간|주간] (기본: 월간) — 챕터 뷰 삭제됨
- Google Calendar 스타일 스패닝 카드 (월~금 5일만, 토/일 제외)
- 캘린더 셀 내부에 일정/task 텍스트 직접 표시
- 오른쪽 패널: 다가오는 업무 (내 Task + 개인 일정만)
- 모든 일정 데이터: `UnifiedSchedule` 단일 모델 사용

## 2. Task 상태 전환 정책

### 2.1 상태 종류

| 상태 | 라벨 | 색상 | 설명 |
|------|------|------|------|
| `pending` | 대기 | 회색 | 아직 시작하지 않은 task |
| `in_progress` | 진행중 | 파란색 | 작업 중인 task |
| `pending_review` | 확인요청 | amber | 운영매니저 확인 대기 중 |
| `completed` | 완료 | 연회색 | 완료된 task |
| `overdue` | 지연 | 빨간색 | 기한 초과 task |
| `unassigned` | 미배정 | 점선 | 담당자 미지정 |

### 2.2 체크박스 전환 규칙

**리뷰어가 없는 task:**

```
체크 → 완료
해제 → 진행중
```

- 대기/지연/진행중 → 체크 클릭 시 바로 `완료`로 전환 (중간 단계 없음)
- 완료 → 체크 해제 시 `진행중`으로 복귀

**리뷰어가 있는 task:**

```
체크 → 확인요청 (체크 표시 안 됨, 토스트 알림)
해제 → 진행중
```

- 대기/지연/진행중 → 체크 클릭 시 바로 `확인요청`으로 전환 (중간 단계 없음)
- 확인요청 상태에서 체크 클릭 시: `진행중`으로 복귀 (확인 요청 해제)
- 완료는 리뷰어(운영매니저)만 처리 가능
- 완료되면 체크 표시 + 타이틀 취소선

### 2.3 자동 상태 전환

- **상세 모달 열기**: `대기` → `진행중` 자동 전환

### 2.4 토스트 알림

리뷰어가 있는 task에서 체크박스 클릭 시 카드 상단에 3초간 표시:

> "운영매니저가 확인하면 완료 처리됩니다"

## 3. 확인 요청(Review) 정책

### 3.1 확인 요청 흐름

1. task에 `reviewerId`가 설정되어 있으면 "확인필요" task
2. 학관매가 상세 모달에서 **"확인 요청하기"** 버튼 클릭
3. task 상태가 `pending_review`로 변경
4. 리뷰어(담당 운영매니저)가 자동 설정됨
5. 운영매니저가 확인 후 `completed`로 변경

### 3.2 확인 요청 취소

- 상세 모달에서 **"확인요청 취소"** 버튼으로 취소 가능
- 취소 시 `in_progress`로 복귀

### 3.3 리뷰어 자동 설정

- 학관매가 속한 트랙의 `operator` 정보에서 자동 추출
- 별도로 리뷰어를 선택하는 UI 없음 (담당 운영매니저가 자동 지정)

### 3.4 시스템 안내 메시지

리뷰어가 설정된 task의 상세 모달 채팅 영역 상단에 시스템 안내 표시:

> "운영매니저 확인이 필요한 Task입니다. 확인 요청을 하면 {검토자}님이 검토 후 완료 처리합니다."

## 4. 상세 모달 정책

### 4.1 헤더 레이아웃

- **1행**: 타이틀 + 우선순위뱃지(긴급/중요, 있을 때만) + `|` + 소스뱃지(시스템/요청/이슈) — 카테고리 뱃지는 표시하지 않음
- **2행**: 상태 드롭다운

### 4.1.1 본문 표시 정보

- 담당자, 검토자, 일정, 시간, 생성자, 완료 시각
- 상세 내용, 기대 산출물, 첨부파일
- 채팅 (시스템 안내 + 대화 내역)

### 4.2 상태 변경 드롭다운

헤더의 StatusBadge를 클릭하면 드롭다운이 열리며, 상태를 직접 전환할 수 있다.

| 드롭다운 항목 | 표시 조건 | 동작 |
|--------------|----------|------|
| 대기 | 항상 | `pending`으로 변경 |
| 진행중 | 항상 | `in_progress`로 변경 |
| 확인요청 | 리뷰어가 있는 task만 | 리뷰어 설정 + `pending_review` |
| 완료 | 리뷰어가 없는 task만 | `completed`로 변경 |

- 현재 상태는 "현재" 라벨과 함께 강조 표시
- 완료 상태에서는 드롭다운 비활성화

### 4.3 Footer 영역

- Footer는 채팅 입력만 포함 (상태 변경 버튼 없음)
- 상태 변경은 헤더 드롭다운으로 통합

### 4.4 제한 사항

- 리뷰어가 있는 task는 드롭다운에 "완료" 항목 미표시 → 완료는 리뷰어만 가능
- 리뷰어가 없는 task는 드롭다운에 "확인요청" 항목 미표시
- 완료 상태에서는 드롭다운 열 수 없음

## 5. 카드 표시 정책 (Card variant)

### 5.1 Row 구성

- **Row 1**: [체크박스] [우선순위] 타이틀 | 상태뱃지
- **Row 2**: 시간 · 트랙(선택) · 확인필요 {검토자} — 표시할 내용 없으면 행 미렌더링
- **Row 3**: 소스뱃지 (시스템/요청/요청받은/직접 추가) + 첨부/메시지 아이콘 (오른쪽 정렬). 학관매가 보낸 업무 요청은 "요청" 뱃지로 표시

### 5.2 표시 규칙

- 담당자 이름: 본인 대시보드이므로 **미표시** (`hideAssignee`)
- 트랙 이름: 단일 트랙 내부이므로 기본 미표시 (`showTrack=false`)
- 확인필요: `reviewerId`가 있을 때만 amber 색으로 표시 (이름은 `useNameLookup`으로 참조)
- 완료 시: 타이틀에 취소선만 적용 (카드 전체 opacity 변경 없음)

## 6. 시간 지정 업무 패널

- 선택 날짜의 시간이 지정된 task를 표시
- 09:00 ~ 18:00 시간대별 타임라인
- `startTime` 기준으로 시간대 그룹
- 현재 시각 시간대: 파란색 하이라이트 + 파란색 좌측 보더
- 과거 시간대: 연한 텍스트
- 30분 기준 점선 구분

---

## 7. 업무 리스트 패널

### 7.1 분류

선택 날짜의 시간 미지정 task를 `source` 기반으로 4가지 그룹으로 분류하고, 각 그룹 사이에 구분선(`border-t`)을 표시한다.

| 순서 | 그룹명 | 조건 | 설명 |
|------|--------|------|------|
| 1 | 요청한 | `source === 'request_sent'` | 내가 운영매니저에게 요청한 업무 |
| 2 | 받은 요청 | `source === 'request_received'` | 운영매니저가 나에게 배정한 업무 |
| 3 | 자동 배정 | `source === 'system'` | 시스템이 자동 배정한 업무 |
| 4 | 내가 추가한 | `source === 'self'` | 내가 직접 추가한 개인 업무 |

### 7.2 카드 구성

- 시간 지정 업무 패널과 동일한 `card` variant 사용
- 체크박스, 상세 모달, 상태 전환 규칙 모두 동일
- 담당자 미표시 (`hideAssignee`)
- 시간 정보 없음 (시간 미지정 task)

### 7.3 task 추가

- 패널 자체의 추가 버튼은 제거됨. 헤더의 "업무 생성" 버튼으로 통합 (11장 참조)
- "내 할 일"로 추가한 task → "내가 추가한" 그룹
- "업무 요청"으로 추가한 task → "요청한" 그룹 (`pending_review` 상태로 생성)

## 8. 일정 탭

> 일정 탭의 목적: **"내가 앞으로 해야하는 일이 뭐지?"** 에 대한 대답.
> 월간/주간 캘린더를 통해 앞으로 해야할 일 + 트랙 흐름에 대한 감각을 바로 심어준다.
> 2-column 레이아웃(65:35)으로 구성된 독립 탭.

### 8.1 레이아웃

| 뷰 | 레이아웃 | 설명 |
|-----|---------|------|
| 월간 | 캘린더 65% + 오른쪽 패널 35% | 셀 클릭 → 오른쪽 미니 시간 그리드로 상세 표시 |
| 주간 | 캘린더 **100% 풀 너비** | 오른쪽 패널 없음 — 시간 그리드 자체가 충분한 정보 제공 |

### 8.2 데이터 소스 및 정보 레이어

| # | 레이어 | Hook 필드 | 데이터 타입 | 표시 위치 |
|---|--------|----------|-----------|----------|
| 1 | 챕터 | `chapters` | `UnifiedSchedule (type: 'chapter')` | All-day 스패닝 라인 |
| 2 | 커리큘럼 | `trackSchedules` (curriculum) | `UnifiedSchedule (type: 'curriculum')` | All-day 스패닝 카드 |
| 3 | 운영일정 | `operationPeriods` | `UnifiedSchedule (type: 'operation_period')` | All-day 스패닝 카드 |
| 4 | 기간 Task | `periodTasks` | `TrackTask (endDate !== scheduledDate)` | All-day 스패닝 바 (dot + 제목 + 날짜) |
| 5 | 일반 Task | `pointTasks` | `TrackTask (!endDate)` | 주간: 시간 그리드 블록, 월간: dot |
| 6 | 개인 일정 | `personalSchedules` | `UnifiedSchedule (type: 'personal')` | Task와 동일 위치 |

> **기간 Task와 일반 Task는 동일한 source 구분 원칙 적용**: 할 일=■네모(blue), MY(내것)=●원(teal). 기간 Task는 여러 날에 걸쳐 이어지는 바 형태로 표시되며, 카드에 날짜 범위도 함께 표시
> **필터는 source 기준 2개**: "할 일"(시스템/매니저 배정) / "MY"(내가 만든 것 + 개인 일정)로 구분. 범례 필터 토글로 표시/숨김 가능

### 8.3 Task 필터 규칙

```
캘린더에 표시하는 Task (calendarTasks):
  - type !== 'daily' (매일 반복 제외 — 업무탭 시간패널에서 이미 표시)
  - status !== 'completed'
  - !description?.startsWith('__self_added__') (내가 추가한 단일 업무는 업무탭에서 표시)

기간 Task (periodTasks) = calendarTasks 중 endDate && endDate !== scheduledDate
일반 Task (pointTasks) = calendarTasks 중 !endDate || endDate === scheduledDate
```

### 8.4 뷰 전환 정책

| 순서 | 뷰 | 기본 | 설명 |
|------|-----|:----:|------|
| 1 | 월간 | O | 큰 그림 파악. 챕터/커리큘럼/운영일정 중심, Task는 간소화 |
| 2 | 주간 | | 상세 확인. **시간 그리드**(09~18시, 24px/슬롯) + Task 블록 배치 |

- 뷰 전환 토글: 캘린더 헤더 **왼쪽** `[월간|주간]` + **범례 필터 토글** (노션 스타일 체크박스로 레이어 표시/숨김)
- 이동 네비게이션(< 현재 >): 캘린더 헤더 **오른쪽** (항상 자리 차지, offset=0이면 투명)
- 기본 뷰: **월간**
- `view` state는 부모(`StaffPage`)에서 관리 → 월간/주간에 따라 레이아웃(2col/1col) 자동 전환

### 8.5 주간 뷰 (Google Calendar 스타일)

#### 구조

```
┌──────────────────────────────────────────────────┐
│ 날짜 헤더 (월9, 화10, 수11, 목12, 금13)             │
├──────────────────────────────────────────────────┤
│ All-day 영역 (스패닝 바)                           │
│  ① 챕터: 얇은 라인 (grayscale)                     │
│  ② 커리큘럼: 얇은 카드 (muted blue 순환)           │
│  ③ 운영일정: 얇은 카드 (muted violet)              │
│  ④⑤ 기간 Task: 작은 바 (시스템=gray, 내것=teal)   │
├────────┬──────┬──────┬──────┬──────┬──────┤
│ 시간축  │ 월   │ 화   │ 수   │ 목   │ 금   │
│ 09:00  │      │      │      │      │      │
│ 09:30  │      │      │      │      │      │
│  ...   │ Task │ Task │ Task │ Task │ Task │
│ 17:30  │      │      │      │      │      │
└────────┴──────┴──────┴──────┴──────┴──────┘
```

#### 시간 그리드

- 09:00 ~ 18:00, 30분 단위 (18 슬롯)
- 각 슬롯 높이: **24px** → 총 ~432px (컴팩트, 스크롤 최소화)
- 정시: 실선 (`border-foreground/[0.05]`)
- 30분: 점선 (`border-dashed border-foreground/[0.03]`)
- 시간 라벨: 좌측 52px 컬럼, `text-[10px] text-foreground/25`

#### Task 블록 배치

- `startTime` 기준으로 y좌표 계산
- `endTime`이 있으면 높이 결정, 없으면 기본 **1슬롯**(24px)
- **카드 디자인**: 흰 배경 + 얇은 테두리 + 미세 그림자 (`bg-background border-foreground/[0.08] shadow-[0_0.5px_2px]`)
- ■/○ dot으로 할 일/MY 구분, 텍스트 `text-[11px] font-medium text-foreground/70`
- 30분 카드: 제목(truncate) + 오른쪽 시간. 1시간+ 카드: 상단 제목 + 하단 시간 범위
- **겹침 처리**: 같은 시간대 복수 Task는 가로로 나눠서 배치 (각 아이템별 겹침 그룹 크기 계산)
- 완료 Task: `opacity-30` + 취소선
- **클릭**: Task 카드 클릭 시 TaskDetailModal 열림 (업무 탭과 동일)

#### 시간 미지정 Task

- 시간 그리드 **안** 상단에 sticky 행으로 표시 (스크롤바 폭 공유 → 세로 그리드 정렬)
- ■/○ dot + 제목, 최대 3개 + 오버플로 카운트
- 좌측 "미지정" 라벨

#### 현재 시간 표시

- 오늘 컬럼: `bg-blue-500/[0.015]` 배경
- 현재 시간 라인: 빨간 가로선 + 좌측 빨간 원형 dot

#### 주간 뷰 레이아웃

- **풀 너비** (오른쪽 패널 없음) — 시간 그리드가 넓게 표시되어 가독성 극대화

### 8.6 월간 뷰

#### 구조 (주별 행)

각 주 행 내부에 All-day 스패닝 바 + 날짜 셀:

```
┌──────────────────────────────────────┐
│ ─── 챕터 3: 심화 기술 ────────────── │  ← chapter line
│ ▓ 프로젝트 관리 심화 ▓│▓ 인메모리 ▓▓ │  ← curriculum bars
│ ░░ 면담 주간 ░░░░░░░░░░░░░░░░░░░░░ │  ← operation period
│ ■ 포트폴리오 준비 ■■■│              │  ← period task bar
│  9  │ 10  │ 11  │ 12  │ 13  │       ← date cells
│ ·출석│ ·면담│ ·보고│     │     │       ← task dots (daily 제외)
└──────────────────────────────────────┘
```

- 7개 레이어 모두 표시하되 간소화
- 기간 Task: 10px 높이 작은 바
- 일반 Task: dot(1px) + 제목(7px) (최대 2개, +N 오버플로)
- 오늘 셀: `bg-blue-500` 원형 날짜 번호

### 8.7 색상 시스템 (LAYER_STYLES)

| 레이어 | 배경 | 텍스트 | 형태 |
|--------|------|--------|------|
| 챕터 | `foreground/[0.04]` | `foreground/50` | 얇은 가로선 + 텍스트 |
| 커리큘럼 | `blue-500/[0.12]` | `blue-800` (semibold) | 채움 바 `rounded-[4px]` |
| 운영일정 | `violet-500/[0.08]` | `violet-700` | 채움 바 + 얇은 테두리 `border-violet-500/15` |
| 기간 Task (할 일) | `blue-500/[0.08]` | `blue-800/70` | ■ dot + 제목 + 날짜 범위. 여러 날 스패닝 바 |
| 기간 Task (MY) | `teal-500/[0.08]` | `teal-800/70` | ● dot + 제목 + 날짜 범위. 여러 날 스패닝 바 |
| 일반 Task (할 일) | `blue-500/[0.08]` | `foreground/80` | ■ 네모 dot + 둥근 블록 |
| 일반 Task (MY) | `teal-500/[0.08]` | `teal-800` | ● 원형 dot + 둥근 블록 |

> **구분 원칙**: 기간/일반 구분 없이 **source 기준**으로만 색상 구분. 할 일(시스템/매니저)=■blue, MY(내것)=●teal.
> 기간 Task는 일반 Task와 동일한 색상/dot을 사용하되, 여러 날에 걸쳐 이어지는 바 형태 + 날짜 범위 라벨로 시각적 차이를 둠.

### 8.8 월간 vs 주간 일관성

| 정보 | 월간 | 주간 |
|------|------|------|
| 챕터 | 얇은 라인 | 얇은 라인 (동일) |
| 커리큘럼 | 얇은 카드 | 얇은 카드 (동일) |
| 운영일정 | 얇은 카드 | 얇은 카드 (동일) |
| 기간 Task | dot + 제목 + 날짜 스패닝 바 | dot + 제목 + 날짜 스패닝 바 (동일) |
| 일반 Task | dot + 제목 | **시간 그리드 블록** (상세) |

> 기간 Task와 일반 Task 모두 동일한 ■/● dot 구분 사용. 기간 Task는 여러 날에 걸치는 바 형태로 자연스럽게 구분됨.
> 차이는 일반 Task 표현 방식에서만 발생 (월간: dot, 주간: 시간 블록).

### 8.9 범례 필터 토글

헤더 왼쪽에 `[월간|주간]` 토글 옆으로 범례를 배치. 각 범례 앞에 **노션 스타일 체크박스** (11x11px, `rounded-[2px]`):

| 범례 | 아이콘 | 색상 | 클릭 시 |
|------|--------|------|---------|
| 커리큘럼 | 직사각형 바 | `blue-500/[0.12]` | All-day 커리큘럼 바 표시/숨김 |
| 운영일정 | 직사각형 바 + 테두리 | `violet-500/[0.08]` | All-day 운영일정 바 표시/숨김 |
| 할 일 | ■ 네모 | `blue-500` | 시스템/매니저 배정 Task (기간+일반) 표시/숨김 |
| MY | ● 원 | `teal-500` | 내가 만든 Task + 개인 일정 표시/숨김 |

- 체크박스 활성: `border-foreground/35` + 체크 아이콘(`text-foreground/45`)
- 체크박스 비활성: `border-foreground/15`, 색상 아이콘 `opacity-30`, 텍스트 `text-foreground/25`
- 활성 시 범례 텍스트: `font-medium text-foreground/70` (선명하게)
- 챕터 라인은 항상 표시 (필터 대상 아님)

### 8.10 오른쪽 패널 — 선택 날짜 미니 시간 그리드 (월간 뷰 전용)

월간 캘린더 셀 클릭 시 **오른쪽 패널이 해당 날짜의 미니 시간 그리드를 표시**.
주간 뷰에서는 오른쪽 패널 없음 (풀 너비).

| 영역 | 내용 |
|------|------|
| 헤더 | 선택 날짜 (기본: 오늘) + "추가" 버튼 |
| 기간 할 일 (상단) | 해당 날짜에 걸쳐 있는 기간 Task (■gray=시스템, ●teal=내것). 날짜 범위 + **D-day 라벨** 표시 |
| 미니 시간 그리드 | 09~18시, 슬롯 20px. 주간 캘린더와 동일한 카드 디자인. **겹침 처리** 지원 |
| 시간 미지정 (하단) | ■/● dot + 제목 리스트 |
| 일정 추가 폼 | 제목(필수) + 시간(선택) 입력 |

> Task 카드 클릭 시 TaskDetailModal 열림 (주간 캘린더, 업무 탭과 동일).

**기간 할 일 D-day 표시 정책**:
- 선택 날짜 기준으로 기간 Task의 종료일까지 남은 일수를 `D-N` 형식으로 표시
- 종료일 당일이면 `D-Day` 표시
- D-2 이하(D-2, D-1, D-Day)일 때 빨간색(`destructive`) 배경+텍스트로 긴급 강조
- D-3 이상이면 회색 배경의 일반 라벨로 표시
- 라벨 위치: 카드 우측 끝, 날짜 범위는 카드 하단에 별도 표시
> 주간 캘린더와 동일한 겹침 처리 알고리즘 적용.

### 8.11 개인 일정 정책

| 항목 | 설명 |
|------|------|
| 데이터 모델 | `UnifiedSchedule { type: 'personal', source: 'manual', creatorId, startDate, startTime?, description? }` |
| 추가 | 오른쪽 패널 헤더 "일정 추가" 버튼 → 입력 폼 (제목 필수, 시간 선택) |
| 삭제 | 개인 일정만 삭제 가능 (hover 시 삭제 버튼) |
| 색상 | teal 계열 고정 (● 원형 dot) |
| 범위 | 본인만 확인 가능 (운영매니저에게 미노출) |
| 저장소 | `useAdminStore.schedules` (UnifiedSchedule 단일 배열, `type: 'personal'`로 필터) |

### 8.12 운영일정 (operation_period) 정책

| 항목 | 설명 |
|------|------|
| 데이터 모델 | `UnifiedSchedule { type: 'operation_period', source: 'manual', trackId, creatorId }` |
| 생성 권한 | 운기매/운영매만 생성 가능 (학관매는 읽기만) |
| 예시 | "A 과제 주간", "면담 주간", "중간평가 주간", "코드리뷰 기간" |
| `track_event`과의 차이 | `track_event`은 단발성 이벤트(멘토링 데이, 팀빌딩), `operation_period`는 기간형 운영 페이즈 |
| 색상 | violet 계열 (`OP_PERIOD_PALETTE`) |

## 9. 소통 채널 정책

> 업무 탭 오른쪽 50% 영역에 배치되는 실시간 소통 채널

### 9.1 레이아웃 구조

소통 채널은 **하나의 rounded 카드 컨테이너** 안에 **헤더 탭 + 채팅 영역**으로 구성된다.

| 영역 | 설명 |
|------|------|
| 헤더 탭 | `[알림] [이운영] [이학관] [박학관]` 가로 탭 버튼. 활성 탭 `bg-foreground/[0.06]`, unread 뱃지 |
| 채팅 영역 | 메시지/알림 피드 + 입력창 |

> 기존 80px 사이드바 → 헤더 탭으로 이동. 확보된 공간으로 업무 탭 좌측 패널 비율 확대 (25:25:50).

- 전체 컨테이너: `rounded-xl border border-foreground/[0.08] bg-background shadow-[0_1px_4px_rgba(0,0,0,0.06)]`
- 사이드바와 채팅 영역은 `border-r border-foreground/[0.06]`로 내부 구분
- 사이드바 배경: `bg-foreground/[0.015]` (채팅 영역보다 살짝 어두운 톤)

### 9.2 사이드바 — 알림 영역 (상단)

- 사이드바 최상단에 **알림(🔔) 탭**을 배치 (`h-11`, 채팅 헤더와 동일 높이)
- 채팅 헤더 하단 `border-b`와 사이드바 알림 영역 하단 `border-b`가 수평으로 정렬됨
- Bell 아이콘 + "알림" 텍스트로 구성
- unread 뱃지: 미확인 알림 수 표시 (빨간 rounded-[4px] 뱃지)
- 클릭 시 채팅 영역이 **시스템 알림 피드**로 전환

### 9.3 사이드바 — 대화 상대 목록 (하단)

| 대화 상대 | 유형 | 설명 |
|-----------|------|------|
| 운영매니저 | `operator` | 담당 운영매니저와의 1:1 채널 (기본 선택) |
| 같은 트랙 학관매 | `peer` | 동일 트랙에 배정된 다른 학관매와의 1:1 DM |

- 이름(`text-sm`, `truncate whitespace-nowrap` — 절대 줄바꿈 안됨) + 역할 라벨(`text-[10px]`)
- 활성 상대: `bg-background shadow-[0_1px_2px]`로 부각
- 비활성: `font-semibold text-foreground/40`
- unread > 0: 버튼 우상단에 빨간 rounded 뱃지 (`absolute -right-0.5 -top-0.5`)
- 대화 상대 사이: 가벼운 구분선 (`mx-2.5 my-1.5 h-px bg-foreground/[0.04]`)
- 클릭 시 해당 대화방으로 전환, reply target 초기화

### 9.4 시스템 알림 피드

- "알림" 탭 선택 시 채팅 영역 대신 **알림 피드** 표시
- 입력창 없음 (읽기 전용)
- 날짜별 그룹핑 (DateDivider)

**알림 유형 및 아이콘:**

| 유형 | 아이콘 | 색상 | 예시 |
|------|--------|------|------|
| `task_assigned` | ListTodo | blue | 새 업무가 배정되었습니다 |
| `task_completed` | CheckCircle2 | emerald | 업무가 완료되었습니다 |
| `task_overdue` | AlertTriangle | amber | 마감 시간이 지났습니다 |
| `review_requested` | UserCheck | violet | 확인 요청이 등록되었습니다 |
| `review_done` | CheckCircle2 | emerald | 매니저가 확인을 완료했습니다 |
| `reminder` | Clock | orange | 30분 후 시작 예정입니다 |
| `general` | Bell | gray | 일반 안내 메시지 |

- task 연결된 알림: 클릭 시 TaskDetailModal 열기
- task 없는 알림: 클릭 비활성

### 9.5 공지 메시지 (Pinned)

- 운영매니저 채널의 pinned 메시지를 **모든 채팅/알림 탭에서 공통 표시**
- 채팅 헤더 바로 아래에 Pin 아이콘과 함께 1줄 표시
- `border-y border-foreground/[0.06] bg-foreground/[0.015]`

### 9.6 메시지 유형

| 유형 | 표시 방식 | 대화방 |
|------|----------|--------|
| 일반 메시지 | 말풍선 (좌: 상대, 우: 나) | 운영매/학관매 모두 |
| 공지 메시지 | 중앙 정렬 pill 형태 | 운영매 채널만 |
| Task 스레드 미리보기 | 카드 형태 (아래 9.7 참조) | 운영매 채널만 |

### 9.7 Task 스레드 미리보기

- 운영매니저 대화방에서 task 관련 대화를 카드 형태로 표시
- **카드 구성**:
  - 1행: ListTodo 아이콘 + **task 제목** (`text-[12px] font-semibold`) + 안읽음 뱃지(파란 rounded) + "→"
  - 2행: 마지막 발화자(`font-medium`) + 마지막 메시지 내용 (들여쓰기)
- **읽음/미읽음 상태**:
  - 미읽음: `border-blue-500/15 bg-blue-50/80` + 파란 아이콘 + 안읽음 카운트 뱃지
  - 읽음: `border-foreground/[0.05] bg-foreground/[0.015]` + 회색 아이콘 + 뱃지 미표시
- 클릭 시: 읽음 처리(`readThreadIds`에 추가) + TaskDetailModal 열기

### 9.8 안 읽은 스레드 추적

- `readThreadIds` (Set): 클릭한 스레드의 taskId를 저장
- 미읽음 스레드가 있으면 입력창 위에 **파란 배너** 표시: "안 읽은 스레드 N개 ↑"
- 배너 클릭 시: 첫 번째 미읽음 스레드 카드로 `scrollIntoView` (smooth, center)

### 9.9 메시지 그룹핑

- **날짜 구분선**: "오늘", "어제", "M월 D일 요일" 형태로 날짜가 바뀔 때 표시
- **시간 갭**: 같은 발신자의 연속 메시지라도 5분 이상 간격이 있으면 시간 다시 표시
- **연속 메시지**: 같은 발신자의 연속 메시지는 이름 생략 + 말풍선 라운드 조정

### 9.10 메시지 인터랙션

- **호버 액션바**: 메시지 위에 마우스 올리면 상단에 [좋아요] [답장] [더보기] 버튼 표시
- **답장(Reply)**: 답장 클릭 시 입력창 상단에 원본 메시지 미리보기 표시, 전송 시 답장 인용 포함
- **리액션**: 좋아요 버튼 클릭 시 메시지 하단에 이모지 + 카운트 뱃지 표시

### 9.11 입력 영역

- 채팅 채널에서만 표시 (시스템 알림 탭에서는 숨김)
- 멀티라인 textarea (자동 높이 조절, 최대 80px)

- 전송 버튼: 텍스트 있을 때만 활성화, 클릭 시 `active:scale-90` 애니메이션

## 11. 업무 생성 정책

### 11.1 접근 방식

- 헤더 우측 "업무 생성" 버튼 클릭 시 모달 표시
- 업무/일정 탭에서만 버튼 표시 (면담 탭에서는 숨김)
- 모달 내부에 **[내 할 일 | 업무 요청]** 2개 탭으로 구분

### 11.2 탭 1: 내 할 일

나만 보이는 개인 업무를 생성한다.

**안내 문구**: "나만 볼 수 있는 개인 업무입니다. 직접 추가하고 관리할 수 있습니다."

| 필드 | 필수 | 기본값 | 설명 |
|------|:----:|--------|------|
| 제목 | O | — | 업무 제목 |
| 시작일 | O | 현재 선택 날짜 | date input |
| 종료일 | — | 현재 선택 날짜 | date input, min=시작일 |
| 시간 | — | 미지정 | 기본은 "시간 미지정" 상태. 클릭 시 time input 전환. "미지정으로 변경" 링크로 해제 가능 |
| 설명 | — | — | textarea (3줄) |

**생성 결과**:
- `type: 'manual'`, description 마커: `'__self_added__'` (설명이 있으면 `'__self_added__\n{설명}'`)
- `source`는 "내가 추가한"(`self`)으로 분류
- 시간 미지정 시 마감 업무 패널에 표시, 시간 지정 시 시간 지정 업무 패널에 표시

**기간 일정 생성 시**:
- 종료일이 시작일과 다른 경우 **기간 업무(period task)** 로 생성
- 생성 완료 후 sonner 토스트 알림: "기간 일정은 일정 탭에서 확인할 수 있습니다"
- 토스트에 "일정 탭으로 이동" 액션 버튼 포함 → 클릭 시 일정 탭으로 전환

### 11.3 탭 2: 업무 요청

운영매니저에게 업무를 요청하거나 이슈를 보고한다.

**안내 문구**: "운영매니저에게 업무를 요청합니다. 확인요청 상태로 전달됩니다."

| 필드 | 필수 | 기본값 | 설명 |
|------|:----:|--------|------|
| 제목 | O | — | 요청 제목 |
| 내용 | O | — | 요청 내용 (textarea 4줄) |
| 긴급 요청 | — | OFF | 체크박스 토글. 활성 시 빨간색 강조 |

**생성 결과**:
- `type: 'manual'`, description 마커: `'__request_sent__\n{내용}'`
- `source`는 "요청"(`request_sent`)으로 분류
- `status`: 바로 `pending_review` 상태로 생성
- `reviewerId`: 학관매가 속한 트랙의 운영매니저가 자동 지정
- `scheduledDate`: 현재 선택 날짜
- 긴급 요청 시 시스템 메시지("⚠️ 긴급 요청입니다") 자동 첨부
- 생성 완료 후 sonner 토스트: "업무 요청이 운영매니저에게 전달되었습니다"

**요청한 task의 표시**:
- 마감 업무 패널 최상단 "요청한" 그룹에 표시 (amber `확인요청` 상태뱃지)
- 카드 소스뱃지: "요청" 표시
- 운영매니저가 확인 후 `completed`로 변경하면 완료 처리

### 11.4 description 마커 시스템

TrackTask에는 `source` 필드가 없으므로, `description` 필드의 접두사(prefix)로 source를 구분한다.

| 마커 | source 매핑 | 설명 |
|------|------------|------|
| `'__self_added__'` 또는 `'__self_added__\n...'` | `self` | 학관매가 직접 추가한 개인 업무 |
| `'__request_sent__'` 또는 `'__request_sent__\n...'` | `request_sent` | 학관매가 운영매니저에게 보낸 요청 |
| 마커 없음 + `type === 'manual'` | `request_received` | 운영매니저가 배정한 업무 |
| 마커 없음 + `type !== 'manual'` | `system` | 시스템 자동 배정 업무 |

- `task-adapter.ts`의 `mapSource()` 함수가 마커 기반으로 source를 결정
- `cleanDescription()` 함수가 마커를 제거하고 실제 사용자 입력 내용만 반환
- UI에서는 마커가 보이지 않음
