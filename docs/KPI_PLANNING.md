# 중간성과 KPI 기획

> 최종 갱신: 2026-03-05
> 상태: 기획 중 (다음주 수요일 리더십 미팅 전까지 구체화 필요)

---

## 배경

### 원래 프로젝트 목적

- 트랙을 늘려도 운영기획매니저(총괄)를 추가 채용하지 않는 것
- 총괄 1명이 더 많은 트랙을 맡을 수 있도록, 업무완료율·이슈처리율만 보고 트랙 운영 상태를 신뢰할 수 있는 구조
- 최소 품질 지표(NPS, 만족도, 이탈률)를 유지하면서 운영 효율화

### 예상 못한 변수

- 중간에 **운영매니저(계약직)** 직책이 추가됨
- 시스템 없이도 운영매니저 추가만으로 총괄의 업무가 줄어드는 구조
- → 시스템의 고유 기여를 분리해서 증명해야 하는 과제 발생

### 파일럿 일정

- **4월부터** 3개 트랙에 시스템 선적용
- 약 1개월 운영 후 중간성과 측정 예정

---

## KPI 프레임워크: 3단계 인과 체인

```
[1단계] Task 커버리지가 충분한가?  (시스템의 기본 자격)
    ↓ Yes
[2단계] 그래서 뭐가 좋아졌는가?   (운영 효율성 효과)
    ↓
[3단계] 최종 품질에 영향을 주는가? (가드레일)
```

### 1단계: Task 관리 커버리지 (전제조건, 우선 측정)

> "학관이 하는 일의 몇 %가 이 시스템으로 관리되는가?"

- **정의**: `시스템 Task / (시스템 Task + self-added Task + 시스템 밖 업무)`
- **목표**: 80% 이상이면 시스템으로서 충분하다고 판단
- **측정 시점**: 파일럿 1~2주차에 빠르게 확인
- **측정 방법**:
  - 학관에게 "시스템에 없는 업무를 했을 때 기록" 요청
  - 시스템 내 `source: 'self'` Task 비율 추적 (별도 개발 불필요)
- **커버리지가 낮을 경우**: 성과 측정 이전에 템플릿 보강이 우선 과제. "파일럿을 통해 누락된 업무 유형을 발견하고 반영했다"도 유의미한 중간 산출물

### 2단계: 운영 효율성 효과 (커버리지 확인 후 측정)

| 지표 | 측정 방법 | 비고 |
|------|----------|------|
| **소통 비용 감소** | 운영매/총괄의 학관 소통 시간·횟수 비교 (파일럿 vs 비파일럿) | 업무 지시·확인·보고가 시스템으로 대체 |
| **업무 누락 감소** | overdue rate, 미완료 Task 비율 | 시스템 리마인드 효과 |
| **상태 파악 속도** | 총괄/운영의 트랙 상태 파악 체감 시간 | 대시보드 vs 구두 확인 |

- 소통시간 측정 시 가장 강력한 방법: **파일럿 트랙 vs 비파일럿 트랙 비교** (자연스러운 대조군)
- 운영매니저에게 "학관과 업무 관련 소통한 시간/횟수를 간단히 기록" 요청하는 일지 방식이 현실적

### 3단계: 최종 품질 지표 (가드레일)

| 지표 | 비고 |
|------|------|
| NPS | 기존 서베이 |
| 수강생 만족도 | 기존 서베이 |
| 이탈률 | 기존 데이터 |

- 1개월 파일럿으로 유의미한 변화 보기 어려움
- "최소한 나빠지지 않았다"를 확인하는 용도

## 리더십 보고 프레이밍

운영매니저 추가를 교란 변수가 아닌 시너지로 프레이밍:

> "운영매니저를 추가했더라도, 시스템 없이는 운영매니저의 업무도 표준화되지 않고 총괄의 가시성도 확보되지 않는다. 이 시스템은 운영매니저가 제대로 일하게 하는 인프라다."

보고 스토리:

> "먼저 시스템이 학관 업무를 충분히 커버하는지 검증했습니다 (1단계).
> 커버리지가 충분하다는 걸 확인한 후, 그로 인해 운영 소통 비용이 실제로 줄었는지를 측정했습니다 (2단계).
> 그리고 이 과정에서 수강생 품질 지표는 유지되었습니다 (3단계)."

## 진단 매트릭스

| 상황 | 의미 | 대응 |
|------|------|------|
| 커버리지 높음 + 소통시간 줄었음 | 시스템이 제대로 작동 | 성과 보고 |
| 커버리지 높음 + 소통시간 안 줄었음 | 시스템은 좋은데 안 쓰고 있음 | adoption/UX 문제 해결 |
| 커버리지 낮음 + 소통시간 줄었음 | 시스템 부족해도 일부 효과 있음 | 템플릿 보강 후 효과 극대화 |
| 커버리지 낮음 + 소통시간 안 줄었음 | 시스템 자체가 부족 | 커버리지 올리는 게 최우선 |
